# 🔍 Sensitive File Scanner — Firefox Extension

Расширение для Firefox, которое **мгновенно проверяет веб-сайт** на наличие общеизвестных уязвимых файлов и директорий.  
Предназначено для пентестеров, разработчиков и всех, кто заботится о безопасности.

## 🚀 Возможности

- **Два режима сканирования**:
  - `Проверить сайт` — сканирует корень домена (например, `https://example.com/.env`)
  - `Проверить текущий путь` — сканирует относительно текущей директории (например, если вы на `https://example.com/admin/`, проверит `https://example.com/admin/.git/config`)
- **Более 360 встроенных путей** (и легко расширяется через `wordlist.txt`)
- **Прогресс-бар** с отображением количества проверенных путей и скорости сканирования
- **Цветовая индикация** статусов ответа (зелёный — 200, красный — 500 и т.д.)
- **Фильтрация** неинформативных ответов (404 не показываются)
- **Загрузка списка путей из внешнего файла `wordlist.txt`** — можно редактировать без переустановки расширения
- **Настраиваемая задержка между пакетами** (rate limiting) для снижения нагрузки на целевой сервер
- **Экспорт результатов** в CSV, JSON и HTML форматы
- **Автоматическое обнаружение технологий** (WordPress, Django, Joomla, Drupal, Laravel, 1C‑Bitrix, Magento, Shopify, TYPO3, Ghost, Strapi, ASP.NET, Ruby on Rails, Express.js и др.)
- **Расширенная проверка уязвимостей**: открытый phpinfo.php, статус‑страницы, Swagger/OpenAPI, Spring Boot Actuator, H2 Console и др.
- **Работает полностью локально**, без внешних сервисов
- **Поддержка Firefox** (Manifest V3)

## 📦 Установка
    Откройте Firefox и перейдите на страницу about:debugging
    Нажмите Этот Firefox → Загрузить временное дополнение
    Выберите файл manifest.json из папки расширения
    Расширение появится в панели инструментов. При следующем запуске браузера его не будет — это нормально для временной загрузки. Чтобы установить навсегда, расширение нужно подписать.

### 🔒 Категории проверяемых файлов (из wordlist.txt)
    Git: .git/config, HEAD, логи, хуки, ссылки
    SVN / Mercurial / CVS / Fossil
    Docker: Dockerfile, docker-compose.yml, .docker/config.json
    CI/CD: Jenkinsfile, .gitlab-ci.yml, .travis.yml, bitbucket-pipelines.yml
    Облака: .aws/credentials, .kube/config, gcp-credentials.json, terraform.tfstate
    Конфиденциальные файлы: .env и его вариации, .htpasswd, wp-config.php, typo3conf/LocalConfiguration.php
    1С-Битрикс: /bitrix/.settings.php, /bitrix/php_interface/dbconn.php, /bitrix/admin/ и другие
    Резервные копии: backup.sql, dump.zip, wwwroot.tar.gz
    IDE и редакторы: .vscode/sftp.json, .idea/workspace.xml
    Веб-шеллы: shell.php, c99.php, adminer.php
    Логи: debug.log, error.log, access.log
    Полный список (~360 путей) можно посмотреть в файле wordlist.txt.

### 📝 Настройка списка путей
    Все проверяемые пути хранятся в файле wordlist.txt (в корне расширения). Вы можете редактировать его в любом текстовом редакторе.
    Формат: один путь на строку.
    Поддерживаются комментарии:
        # — строка игнорируется
        // — также игнорируется
    После изменения файла перезагрузите расширение на странице about:debugging (кнопка «Обновить»).
    Если wordlist.txt отсутствует или пуст, расширение автоматически использует встроенный список из 30+ самых критичных путей.

### ⚙️ Дополнительные настройки
    На странице options.html доступны:
    - **Таймаут запроса** (секунды, 1‑30)
    - **Одновременных запросов** (размер пакета, 1‑20)
    - **Задержка между пакетами** (мс, 0‑5000) – полезно для снижения нагрузки на цель и обхода простых защит от брутфорса
    - Включение/выключение категорий путей (Git, Docker, CI/CD, Облака, CMS, Резервные копии, Логи, IDE)

## 📤 Экспорт результатов
    После сканирования можно экспортировать найденные данные:
    - **CSV** – удобно для импорта в Excel или системы отслеживания задач
    - **JSON** – структурированный формат для дальнейшей обработки
    - **HTML** – красивый самодостаточный отчёт с цветовой индикацией и ссылками на проверенные URL

## 🛡️ Как это помогает
    Сканер быстро выявляет типичные ошибки конфигурации, которые могут привести к утечке конфиденциальных данных:
    - Открытые репозитории Git (.git/config, HEAD и т.д.)
    - Файлы окружения (.env, .env.local и т.д.) с секретами
    - Логи отладки и ошибок
    - Резервные копии баз данных и архивы сайтов
    - Конфигурационные файлы CMS и фреймворков
    - Доступ к панелям администрирования и отладки (phpinfo, swagger, actuator и т.д.)
    - Обнаружение режима DEBUG = True в Django‑приложениях
